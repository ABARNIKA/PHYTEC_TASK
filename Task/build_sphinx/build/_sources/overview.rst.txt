Project Overview
================

Project Summary
---------------

This comprehensive documentation covers the complete development journey for implementing LED blink functionality on the **EFR32ZG28B312F1024IM48-A** microcontroller using the **BRD2705A Rev. A02** development board. The project encountered and solved unique challenges related to ARM TrustZone security extensions.

Target Device
~~~~~~~~~~~~~

**EFR32ZG28B312F1024IM48-A**

- ARM Cortex-M33 with TrustZone Security Extension
- 1024KB Flash, 128KB RAM
- Security Management Unit (SMU)
- Z-Wave Gecko Series 2 SoC

Development Board
~~~~~~~~~~~~~~~~~

**BRD2705A Rev. A02**

- Serial Number: 440316719
- Integrated J-Link OB debugger
- LED1 connected to PC8 (Port C, Pin 8)
- USB-powered, 3.3V operation

Key Technical Challenges
------------------------

TrustZone Security Extension
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The primary challenge encountered was the ARM TrustZone Security Extension blocking external GPIO access:

- GPIO registers protected by Security Management Unit (SMU)
- External debuggers lack proper security credentials
- Standard embedded development tools cannot bypass security

.. warning::
   TrustZone Security Extension cannot be bypassed with external tools.
   Silicon Labs' proprietary security keys required for GPIO access.
